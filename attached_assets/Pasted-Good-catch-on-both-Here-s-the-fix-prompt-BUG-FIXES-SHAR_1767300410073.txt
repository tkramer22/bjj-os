| Technical Error | User-Friendly Message |

| User not found | "Incorrect email or password" |
| Wrong password | "Incorrect email or password" |
| Network error | "Connection error. Please try again." |
| Timeout | "Connection timed out. Please try again." |
| Email already exists | "An account with this email already exists" |
| Invalid email format | "Please enter a valid email address" |
| Password too short | "Password must be at least 8 characters" |
| Any other error | "Something went wrong. Please try again." |

IMPORTANT SECURITY NOTE:
Never specify whether email OR password was wrong - always say "Incorrect email or password"
This prevents attackers from knowing if an email exists in the system.

ERROR DISPLAY:
- Red background or red text (subtle, not jarring)
- Clear, readable font
- Appears below the input field or in a toast
- Disappears when user starts typing again

Example implementation:

try {
  await login(email, password);
} catch (error) {
  let userMessage = "Something went wrong. Please try again.";
  
  if (error.code === 'auth/invalid-credential' || 
      error.code === 'auth/wrong-password' ||
      error.code === 'auth/user-not-found') {
    userMessage = "Incorrect email or password";
  } else if (error.code === 'auth/network-request-failed') {
    userMessage = "Connection error. Please try again.";
  } else if (error.code === 'auth/too-many-requests') {
    userMessage = "Too many attempts. Please try again later.";
  }
  
  setErrorMessage(userMessage);
}
