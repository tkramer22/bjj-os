# Emergency Playbook - If Things Go Wrong

## Scenario 1: SMS Codes Not Sending

**Symptoms:** Users not receiving verification codes

**Quick Fix:**
1. Check Twilio dashboard - are messages sending?
2. Check Twilio account balance
3. Check if phone number is verified in Twilio (for trial mode)
4. Check error logs for Twilio errors

**Workaround:**
- Manually verify users in database
- Or pause new signups until fixed

## Scenario 2: Prof. OS Not Responding

**Symptoms:** Chat loads but no answers come back

**Quick Fix:**
1. Check Claude/OpenAI API keys are valid
2. Check API rate limits
3. Check error logs for API errors
4. Check database connection

**Workaround:**
- Show message: "Prof. OS is temporarily unavailable, try again in a few minutes"

## Scenario 3: Videos Not Loading

**Symptoms:** Video recommendations appear but won't play

**Quick Fix:**
1. Check if YouTube URLs are valid
2. Check if embed code is correct
3. Check browser console for errors

**Workaround:**
- Link to YouTube directly as temporary fix

## Scenario 4: Too Many Signups, Database Overload

**Symptoms:** Slow performance, timeouts

**Quick Fix:**
1. Check database connection pool
2. Check server resources
3. Optimize slow queries

**Workaround:**
- Enable "emergency kill switch" to pause new signups
- Fix issues, then re-enable

## Scenario 5: User Reports Bug I Can't Reproduce

**Quick Fix:**
1. Ask for screenshots
2. Ask what device/browser
3. Ask exact steps to reproduce
4. Try on their device type

**Workaround:**
- Thank them, note the bug, fix later if can't solve immediately
