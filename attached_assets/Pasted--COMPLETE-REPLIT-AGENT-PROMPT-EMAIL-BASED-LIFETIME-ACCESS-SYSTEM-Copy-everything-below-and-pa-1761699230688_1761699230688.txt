Create new table called `lifetime_invitations`:

CREATE TABLE IF NOT EXISTS lifetime_invitations (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) NOT NULL,
  invite_token VARCHAR(255) UNIQUE NOT NULL,
  personal_message TEXT,
  reason VARCHAR(100),
  admin_notes TEXT,
  invited_by_admin INT REFERENCES users(id),
  
  status VARCHAR(50) DEFAULT 'pending',
  -- Status options: 'pending', 'completed', 'expired'
  
  created_at TIMESTAMP DEFAULT NOW(),
  expires_at TIMESTAMP NOT NULL,
  used_at TIMESTAMP,
  completed_by_user_id INT REFERENCES users(id)
);

CREATE INDEX idx_lifetime_invite_token ON lifetime_invitations(invite_token);
CREATE INDEX idx_lifetime_invite_email ON lifetime_invitations(email);
CREATE INDEX idx_lifetime_invite_status ON lifetime_invitations(status);

Also update the `users` table to add these columns if they don't exist:

ALTER TABLE users ADD COLUMN IF NOT EXISTS lifetime_access BOOLEAN DEFAULT FALSE;
ALTER TABLE users ADD COLUMN IF NOT EXISTS is_founding_member BOOLEAN DEFAULT FALSE;
ALTER TABLE users ADD COLUMN IF NOT EXISTS invited_by INT REFERENCES users(id);

POST /api/admin/lifetime/invite

Protected by checkAdminAuth middleware.

Request body:
{
  "email": "user@example.com",
  "personalMessage": "optional personal note",
  "reason": "founding_member",
  "adminNotes": "optional internal notes",
  "expirationDays": 14
}

Logic:
1. Validate email and reason are provided
2. Check if email already has lifetime access (reject if yes)
3. Check if there's already a pending invitation for this email (reject if yes)
4. Generate unique invite token using crypto.randomBytes(32).toString('hex')
5. Calculate expiration date (current date + expirationDays)
6. Insert invitation into lifetime_invitations table
7. Generate magic link: https://bjjos.app/lifetime-signup?token={inviteToken}
8. Send email using nodemailer (see email template below)
9. Return success response

Response:
{
  "success": true,
  "message": "Lifetime access invitation sent successfully",
  "invitationId": 123,
  "email": "user@example.com",
  "expiresAt": "2025-11-11T00:00:00.000Z"
}

Public endpoint (no auth required).

Request body:

  "token": "invite_token_here",

  "username": "johndoe",
  "password": "securepassword123"

Logic:
1. Validate invitation token (same checks as endpoint 3)
2. Check if username is available
3. Hash password using bcrypt
4. Create new user with:
   - email from invitation
   - username, display_name, password_hash from request
   - lifetime_access = true
   - is_founding_member = true
   - invited_by = invitation.invited_by_admin
5. Update invitation status to 'completed' and set used_at = NOW()
6. Generate JWT token for new user
7. Return token and user data

Response:

  "success": true,
  "message": "Account created successfully",
  "token": "jwt_token_here",
  "user": {
    "id": 123,
